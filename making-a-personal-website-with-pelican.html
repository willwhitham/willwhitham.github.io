Making a personal website with Pelican

Posted on Wed 05 January 2022 in articles

If you found yourself on this page you were probably here for comparative psychology, not extremely amateur website development. But! If you want a simple and free website, maybe this post will help.

Pelican is a static website generator for Python. 'Static' means roughly what it sounds like - no bells, no whistles, what you see is what you get. Since I won't have a storefront or messageboard or anything that demands more complexity, generating a static website means I can share what is essentially a series of .pdf pages without too much time or expense. I spent a few mornings on this, and hosting the site is free(!!) through github pages. This walkthrough is in Windows, and other platforms should be similar.

Getting started

You'll need some software and working knowledge to get started:

  • Required :: git, a GitHub account, Python (with Pelican package installed), basic terminal know-how.
  • Recommended :: Pycharm IDE (for ease of project management), GitHub Desktop (for version control and synergy with GitHub pages, which will eventually host your site)

Next, open a terminal window wherever pelican installed and run 

.\pelican-quickstart

to get started. The questions are a little vague, see the Pelican documentation for clues. I got by with:

+ Where do you want to create your new web site? (hit Enter)
+ Do you want a URL prefix: Yes
+ Article pagination: No
+ URL prefix: http://{yourusername}.github.io
+ Generate a Fabfile/Makefile: Yes
+ Auto-reload & simpleHTTP script: Yes
+ Upload mechanisms: No for everything except Github Pages
+ Is this your personal page ({yourusername}.github.io)? Yes

Adding content

The hard part is mostly done! Quickstart should have generated a content folder for you - this is where you will stick all of the stuff you're hoping to share with the world. By default Pelican recognizes three bins for your content:

  1. Articles - signed, dated, blog-post-style content like what you're currently reading.
  2. Pages - more free-standing content pages, e.g., 'about this page', 'contact me', etc.
  3. Images

To keep things simple, create folders called articles, pages, and images in your content folder. For articles and pages, Pelican likes two formats: reStructuredText and MarkDown. I use MarkDown in jupyter-notebook, so I'm using it exclusively. To make this post, I created a text file in the articles folder called howto.md. Each article requires a title, followed by any other metadata you choose to include (see Pelican documentation for all the subtitles, authorship, keywords, etc that can be included), then your properly content. Pages are created in the same way, and both articles and pages can be decorated with images from the images folder.

Admire your handiwork

To preview the site, return to the terminal window and run 

pelican content

Because of the site's simplicity, this should go quickly (<10 seconds), and our shiny new .html pages should start showing up in an output folder adjacent to your content folder. Once the site is built, run 

pelican -rl

to get a live version of your site on your machine only. Enter the localhost id in your browser to preview your page! All of your content should be visible and organized according to Pelican's defaults. Close the server with ctrl+c.

Spice things up

Defaults are kind of boring! Check out new pelican-themes and their documentation to spice things up (I'm using the Flex theme). To change your theme, first clone the repository of the theme you like, then open a terminal window and run 

pelican-theme --install {path to the repository you just cloned}

To see the list of themes you've installed, run 

pelican-themes -l

To switch to a theme, run

pelican-themes {name of your theme}

and add the name of your theme to pelicanconf.py, e.g., 

THEME = 'Flex'

Further customization will depend on your theme, so check out its documentation. I find that there are lots of cool Pelican themes, but the number of cool pelican themes that are also well documented is a lot shorter...

With the Flex them, at least, nice sidebar links are incredibly easy to add. In pelicanconf.py, simply specify each logo and link in a 2-item tuple:

SOCIAL = (
      ('linkedin', 'www.linkedin.com/in/willwhitham000'),
      ('google scholar', 'https://scholar.google.com/citations?user=hP_XTKMAAAAJ&hl=en'),
      ('researchgate', 'https://www.researchgate.net/profile/Will-Whitham-2')
    )
